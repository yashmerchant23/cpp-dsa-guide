C++ for DSA

Everything a Java programmer needs to hit the ground running with C++ for competitive programming & interviews.

Java → C++  |  syntax + STL + resources + roadmap
Coming from Java? The logic and OOP concepts are nearly identical. The biggest differences are: manual memory, no garbage collector, :: scope operator, cin/cout instead of Scanner, and the powerful STL.

Pointers & References (No Java Equivalent)
int a = 10;
int* ptr = &a;    // pointer: stores address of a
*ptr = 20;        // dereference: a is now 20

int& ref = a;     // reference: alias for a
ref = 30;         // a is now 30

// Pass by reference (modify original — Java can't do this for primitives)
void swap(int& x, int& y) { int t = x; x = y; y = t; }

// Dynamic memory (use sparingly in DSA; prefer vectors)
int* arr = new int[5];
delete[] arr;     // MUST free! No garbage collector

STL is your superpower. It's the #1 reason to switch to C++ for CP. Java doesn't have anything as tight. Master these 8 containers and you can solve 90% of DSA problems.
Container Quick Reference
C++ STL Java Equivalent Use Case Key Operations

Binary Tree Node
struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

// Inorder traversal
void inorder(TreeNode* root) {
    if (!root) return;
    inorder(root->left);
    cout << root->val << " ";
    inorder(root->right);
}

// BFS / Level Order
queue<TreeNode*> q;
q.push(root);
while (!q.empty()) {
    TreeNode* node = q.front(); q.pop();
    cout << node->val << " ";
    if (node->left)  q.push(node->left);
    if (node->right) q.push(node->right);
}
Graph (Adjacency List)
int n = 5;  // number of nodes
vector<vector<int>> adj(n + 1);   // 1-indexed
adj[1].push_back(2);               // edge 1 → 2
adj[2].push_back(1);               // undirected

// Weighted: vector of pairs
vector<vector<pair<int,int>>> wadj(n + 1);
wadj[1].push_back({2, 5});          // edge to node 2, weight 5

// BFS
vector<bool> visited(n + 1, false);
queue<int> q;
q.push(1); visited[1] = true;
while (!q.empty()) {
    int u = q.front(); q.pop();
    for (int v : adj[u]) {
        if (!visited[v]) {
            visited[v] = true;
            q.push(v);
        }
    }
}

// DFS
void dfs(int u, vector<vector<int>>& adj, vector<bool>& vis) {
    vis[u] = true;
    for (int v : adj[u])
        if (!vis[v]) dfs(v, adj, vis);
}
Dynamic Programming Template
// 1D DP — Fibonacci
vector<int> dp(n + 1, -1);  // memoization

int fib(int n, vector<int>& dp) {
    if (n <= 1) return n;
    if (dp[n] != -1) return dp[n];
    return dp[n] = fib(n-1, dp) + fib(n-2, dp);
}

Sliding Window
// Fixed window of size k
int windowSum = 0, maxSum = 0;
for (int i = 0; i < n; i++) {
    windowSum += arr[i];
    if (i >= k) windowSum -= arr[i - k];
    if (i >= k - 1) maxSum = max(maxSum, windowSum);
}

// Variable window (two pointers)
int l = 0, ans = 0;
unordered_map<int,int> freq;
for (int r = 0; r < n; r++) {
    freq[arr[r]]++;
    while (freq.size() > k) { // shrink window
        freq[arr[l]]--;
        if (freq[arr[l]] == 0) freq.erase(arr[l]);
        l++;
    }
    ans = max(ans, r - l + 1);
}
Binary Search Template
// Standard binary search on sorted array
int lo = 0, hi = n - 1, ans = -1;
while (lo <= hi) {
    int mid = lo + (hi - lo) / 2;  // avoids overflow
    if (arr[mid] == target) { ans = mid; break; }
    else if (arr[mid] < target) lo = mid + 1;
    else hi = mid - 1;
}

// Binary search on answer
int lo = 1, hi = 1e9, ans = -1;
while (lo <= hi) {
    int mid = lo + (hi - lo) / 2;
    if (isValid(mid)) { ans = mid; hi = mid - 1; }  // find minimum valid
    else lo = mid + 1;
}

Common Java → C++ Mistakes to Avoid

Using int instead of long long

Java's long is 64-bit. C++'s long can be 32-bit on some systems. Always use long long for large values. Define #define ll long long.

Forgetting endl vs '\n'

endl flushes the buffer every time — it's 100x slower. Use "\n" instead, especially after adding ios_base::sync_with_stdio(false).

Memory leaks with new/delete

C++ has no garbage collector. Every new needs a delete. In DSA, prefer vector and struct on the stack — much safer and faster.

Comparing size() with int

vector.size() returns size_t (unsigned). If you write v.size() - 1 when size is 0, you get a huge number. Cast: (int)v.size().

Using == on strings (it works, but...)

Unlike Java, s1 == s2 compares content in C++ strings (not references). This is actually better! No need for .equals().

Erasing multiset elements

ms.erase(val) removes ALL occurrences. To remove just one, use ms.erase(ms.find(val)) — a very common interview bug.
