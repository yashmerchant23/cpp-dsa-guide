Sticking With Java

Java for DSA

The complete Collections Framework, syntax patterns, and curated resources for grinding Data Structures & Algorithms in Java — no detours.

Why stay on Java? The Collections Framework is mature, the syntax is verbose but predictable, and most interview platforms support it fully. The only real cost is verbosity — Java needs more lines than C++ for the same logic, and you occasionally hit a TLE on the tightest time-limit CP problems. For interviews, this almost never matters.

Fast I/O (Codeforces / Competitive Programming)
import java.io.*;
import java.util.*;

public class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringBuilder sb = new StringBuilder();

        int n = Integer.parseInt(br.readLine().trim());
        StringTokenizer st = new StringTokenizer(br.readLine());
        int[] arr = new int[n];
        for (int i = 0; i < n; i++) arr[i] = Integer.parseInt(st.nextToken());

        sb.append("result\n");   // build output, don't print directly each time
        System.out.print(sb);
    }
}
// Scanner is ~10x slower than BufferedReader. Avoid Scanner in CP with large input.
CP consensus: Use BufferedReader + StringTokenizer for reading, and StringBuilder for output — never call System.out.println in a loop with large output, it's slow.

int[] arr = new int[5];               // all zeros by default
int[] arr2 = {1, 2, 3};            // literal init
arr.length;                       // NOT a method! No parentheses.

// 2D array
int[][] grid = new int[3][4];      // 3 rows x 4 cols, all 0

Arrays.sort(arr);                 // ascending, in-place
Arrays.toString(arr);             // prints [1, 2, 3]
Arrays.fill(arr, -1);            // fill all with -1
Arrays.copyOf(arr, 10);          // resize/clone
Arrays.asList(1, 2, 3);        // fixed-size List view (can't add/remove!)

Integer[] boxedArr = {3, 1, 2};
Arrays.sort(boxedArr, (a, b) -> b - a);  // descending
Common trap: You cannot pass a comparator to sort a primitive int[] — Java's Arrays.sort only accepts comparators for object arrays. Use Integer[] if you need custom sort order.
Strings & StringBuilder
String s = "hello";
s.length();                  // 5
s.charAt(0);                // 'h'
s.substring(1, 4);          // "ell" — end EXCLUSIVE
s.indexOf("ll");            // 2, or -1 if not found
s.equals("hello");         // use .equals(), NEVER == for content
s.toCharArray();             // char[] for manipulation
Integer.parseInt("42");      // String → int
String.valueOf(42);          // int → String
s.split(",");                // String[] — built-in split, unlike C++!

// Strings are immutable — every concatenation makes a NEW string
String bad = "";
for (int i = 0; i < n; i++) bad += i;  // O(n²) — DON'T do this in loops!

// Use StringBuilder for mutation — this is your mutable string equivalent
StringBuilder sb = new StringBuilder();
sb.append("hello");
sb.reverse();
sb.deleteCharAt(0);
sb.toString();                // convert back to String when done
The == vs .equals() Trap
String a = new String("hi");
String b = new String("hi");

a == b;          // false — compares MEMORY ADDRESS, not content
a.equals(b);    // true  — compares actual CONTENT

// Same trap applies to Integer objects outside the -128..127 cache range
Integer x = 200, y = 200;
x == y;          // false! (outside cached range)
x.equals(y);    // true

Integer p = 100, q = 100;
p == q;          // true (cached -128 to 127, same object reused)
Rule: Always use .equals() for Strings and boxed types (Integer, Long, etc). Only use == for primitives (int, char, boolean).
Lambdas & Comparators
// Functional interface — used everywhere for custom sorting
List<Integer> list = new ArrayList<>(List.of(3,1,4));
list.sort((a, b) -> a - b);          // ascending
list.sort((a, b) -> b - a);          // descending

// More readable with Comparator static methods
list.sort(Comparator.naturalOrder());
list.sort(Comparator.reverseOrder());

// Sort array of pairs (e.g., int[][] edges) by 2nd element
int[][] edges = {{1,5}, {2,3}};
Arrays.sort(edges, (a, b) -> a[1] - b[1]);

// Multi-level sort: by length, then alphabetically
List<String> words = ...;
words.sort(Comparator.comparingInt(String::length).thenComparing(s -> s));
Watch for integer overflow: (a - b) can overflow for very large/small ints. Prefer Integer.compare(a, b) in sort comparators to be safe.
This is your STL. The Java Collections Framework is a must for any developer preparing for technical interviews — master these and you can solve nearly every DSA problem.
Collection Quick Reference
Java Type Backed By Use Case Key Methods
ArrayList<T> Resizable array Dynamic list, fast random access add, get, set, remove, size
ArrayDeque<T> Resizable array Stack AND queue (preferred over both) push, pop, offer, poll, peek
LinkedList<T> Doubly linked list Queue, frequent insert/delete add, poll, offer, peek
PriorityQueue<T> Binary heap Dijkstra, K-th largest, heaps offer, poll, peek
HashSet<T> HashMap internally Fast lookup, visited tracking add, remove, contains — O(1)
TreeSet<T> Red-Black Tree Sorted unique elements, range queries add, ceiling, floor, first, last
HashMap<K,V> Hash table Frequency count, memoization put, get, getOrDefault, containsKey
TreeMap<K,V> Red-Black Tree Key-value pairs sorted in a specific order put, get, firstKey, ceilingKey
LinkedHashMap<K,V> Hash table + linked list Maintains insertion order Same as HashMap, but ordered
ArrayList Examples
List<Integer> list = new ArrayList<>();
list.add(10);
list.add(0, 5);              // insert at index 0
list.get(0);                 // 5
list.set(0, 99);             // overwrite index 0
list.remove(Integer.valueOf(99)); // remove BY VALUE
list.remove(0);              // remove BY INDEX — overload trap!
list.contains(10);
Collections.sort(list);
Collections.reverse(list);

// 2D List (most common in grid/graph problems)
List<List<Integer>> grid = new ArrayList<>();
for (int i = 0; i < 3; i++) grid.add(new ArrayList<>());
Trap: list.remove(5) removes the element at index 5, not the value 5. To remove by value with an Integer list, box it: list.remove(Integer.valueOf(5)).
ArrayDeque — Stack & Queue (Use This, Not Stack)
// As a STACK (LIFO)
Deque<Integer> stack = new ArrayDeque<>();
stack.push(1);
stack.push(2);
stack.pop();             // removes & returns 2 (top)
stack.peek();            // look at top without removing

// As a QUEUE (FIFO)
Deque<Integer> queue = new ArrayDeque<>();
queue.offer(1);            // add to back
queue.offer(2);
queue.poll();             // removes & returns 1 (front)

// As a DEQUE (both ends — for sliding window problems)
Deque<Integer> dq = new ArrayDeque<>();
dq.addFirst(1); dq.addLast(2);
dq.pollFirst(); dq.pollLast();
Stack extends Vector with synchronization overhead — prefer ArrayDeque for better single-threaded performance in modern code. Don't use the old Stack class.
PriorityQueue (Heap)
// Min-heap (default)
PriorityQueue<Integer> minPQ = new PriorityQueue<>();
minPQ.offer(5);
minPQ.offer(1);
minPQ.poll();          // 1 (smallest) — priority queue is a min-heap by default

// Max-heap — flip with reverseOrder
PriorityQueue<Integer> maxPQ = new PriorityQueue<>(Collections.reverseOrder());

// Custom comparator — min-heap of pairs (dist, node), Dijkstra pattern
PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> a[0] - b[0]);
pq.offer(new int[]{0, src});  // {distance, node}
Heads up: PriorityQueue doesn't support direct indexing or efficient random removal, and iteration order is undefined — only peek()/poll() guarantee the min/max element.
HashMap Examples

// Count frequencies — the most common DSA pattern

    freq.put(w, freq.getOrDefault(w, 0) + 1);
}

// Even shorter with merge

// Check existence

// Iterate over entries

    System.out.println(entry.getKey() + ": " + entry.getValue());
}

// computeIfAbsent — for grouping into lists (graph adjacency, anagram groups)

TreeMap / TreeSet (Sorted Structures)
TreeSet<Integer> ts = new TreeSet<>(List.of(5, 1, 3));
ts.first();             // smallest
ts.last();              // largest
ts.ceiling(2);          // smallest element >= 2
ts.floor(2);            // largest element <= 2
ts.higher(3);           // strictly greater than 3
ts.lower(3);            // strictly less than 3

Java has no built-in multiset. Simulate it with TreeMap<Integer, Integer> (value → count) when you need a sorted structure with duplicates.

Binary Tree Node + Traversals
class TreeNode {
    int val;
    TreeNode left, right;
    TreeNode(int val) { this.val = val; }
}

// Recursive inorder
void inorder(TreeNode root, List<Integer> result) {
    if (root == null) return;
    inorder(root.left, result);
    result.add(root.val);
    inorder(root.right, result);
}

// BFS / Level order — uses ArrayDeque as queue
Deque<TreeNode> q = new ArrayDeque<>();
q.offer(root);
while (!q.isEmpty()) {
    TreeNode node = q.poll();
    System.out.print(node.val + " ");
    if (node.left != null)  q.offer(node.left);
    if (node.right != null) q.offer(node.right);
}
Graph (Adjacency List)
int n = 5;
List<List<Integer>> adj = new ArrayList<>();
for (int i = 0; i <= n; i++) adj.add(new ArrayList<>());
adj.get(1).add(2);    // edge 1 → 2
adj.get(2).add(1);    // undirected

// Weighted graph — List of int[] {neighbor, weight}
List<List<int[]>> wadj = new ArrayList<>();

// BFS
boolean[] visited = new boolean[n + 1];
Deque<Integer> queue = new ArrayDeque<>();
queue.offer(1); visited[1] = true;
while (!queue.isEmpty()) {
    int u = queue.poll();
    for (int v : adj.get(u)) {
        if (!visited[v]) { visited[v] = true; queue.offer(v); }
    }
}

// DFS (recursive)
void dfs(int u, List<List<Integer>> adj, boolean[] visited) {
    visited[u] = true;
    for (int v : adj.get(u))
        if (!visited[v]) dfs(v, adj, visited);
}
Dynamic Programming Template
// 1D DP with memoization — Fibonacci
int[] dp;
int fib(int n) {
    if (n <= 1) return n;
    if (dp[n] != -1) return dp[n];
    return dp[n] = fib(n - 1) + fib(n - 2);
}
// init: dp = new int[n+1]; Arrays.fill(dp, -1);

// 2D DP — Knapsack / LCS pattern
int[][] dp = new int[m + 1][n + 1];   // auto-initialized to 0

Sliding Window
// Variable window (two pointers) — most common interview pattern
int l = 0, ans = 0;
Map<Integer, Integer> freq = new HashMap<>();

for (int r = 0; r < arr.length; r++) {
    freq.merge(arr[r], 1, Integer::sum);
    while (freq.size() > k) {   // shrink window
        freq.put(arr[l], freq.get(arr[l]) - 1);
        if (freq.get(arr[l]) == 0) freq.remove(arr[l]);
        l++;
    }
    ans = Math.max(ans, r - l + 1);
}
Binary Search Template
int lo = 0, hi = arr.length - 1, ans = -1;
while (lo <= hi) {
    int mid = lo + (hi - lo) / 2;   // avoids overflow
    if (arr[mid] == target) { ans = mid; break; }
    else if (arr[mid] < target) lo = mid + 1;
    else hi = mid - 1;
}

// Java's built-in binary search (returns index, or -(insertion point) - 1 if absent)
int idx = Arrays.binarySearch(arr, target);

Common Java DSA Mistakes to Avoid

Using == for String/Integer comparison

Always use .equals() for objects. == compares references, not values — a classic bug that silently passes small test cases and fails larger ones.

String concatenation in loops

Strings are immutable — every += inside a loop creates a new object, turning an O(n) operation into O(n²). Use StringBuilder instead.

Forgetting int overflow

Java's int is 32-bit just like C++. Sums of large arrays or products can silently overflow — switch to long when constraints suggest values beyond ~2×10⁹.

Scanner in competitive programming

Scanner is convenient but slow on large inputs. For CP-style problems with big input sizes, use BufferedReader + StringTokenizer instead.

list.remove(int) ambiguity

On a List<Integer>, remove(5) removes by index, not value. To remove the value 5, write remove(Integer.valueOf(5)).

Modifying a list while iterating

Using a for-each loop and calling list.remove() inside it throws ConcurrentModificationException. Use an explicit Iterator with it.remove(), or iterate backwards by index.
